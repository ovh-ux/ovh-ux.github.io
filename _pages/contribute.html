If you're passionate about those subjects, we'll be happy to gather your advice and your help. We'll always do better together.

  • Feel free to discuss with us on our Gitter.
  • Just be cool and respectful with others and respect our Code Of Conduct
  • You have found a bug, or have some improvement idea? Please create an issue. We have a template for this, and please read our guidelines

You're developer and want contribute:

  • Use this configuration file to respect our javascript styleguides
  • Use this configuration file to respect our css, sass styleguides
  • Your code should be tested

Respect this simple commit message template, this tool will help you: cz-ovh-commit

        
            :

Where type can be :

  • feat: A new feature
  • fix: A bug fix
  • chore: Refactoring, changing the build process or auxiliary tools and libraries etc.
  • docs: Documentation writing
  • style: Cosmetic purpose
  • test: Adding missing or correcting tests
  • For the subject, use the present tense ("Add feature" not "Added feature")
  • Should be submitted through GitHub Pull Requests on develop, we also have a template for this.

Now, if you want to go further, sit comfortably at the fireplace, grab your glasses and immerse yourself in our full documentation.
